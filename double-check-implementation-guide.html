IMPLEMENTATION GUIDE

DoubleCheck

אפליקציית וידוא משלוחים — מצלמים קבלה, מסמנים מה הגיע, שולחים דוח פערים בלחיצת כפתור.

📄 גרסה 1.0 📅 מאי 2026 👤 Alon Gabay 🔒 מסמך פנימי
CTO TECHNOLOGY RULES
  1. PWA-first — האפליקציה חייבת לעבוד כ-Installable PWA עם Service Worker, מצב Offline בסיסי, ו-Add to Home Screen בכל פלטפורמה.
  2. Hebrew RTL by default — כל ממשק המשתמש בעברית עם dir="rtl". אסור placeholder באנגלית. תאריכים ומחירים בפורמט ישראלי (₪, DD/MM/YYYY).
  3. OCR via Vision API → JSON only — הקריאה למודל מחזירה JSON מובנה (לא טקסט חופשי) עם schema קשיח של פריטים. אסור parsing של רגקסים על טקסט גולמי.
  4. Excel via SheetJS in-browser — יצירת ה-Excel קורית בצד הלקוח (xlsx.js), לא ב-server. ככה אין עומס שרת ואפשר לעבוד גם offline.
  5. Privacy by default — תמונות קבלות ומיילים נכנסים נמחקים אוטומטית מ-Storage אחרי 30 יום. למשתמשים אנונימיים — שמירה רק ב-IndexedDB מקומית.
  6. 📨 Email security mandatory — כל webhook נכנס מ-Cloudflare → Vercel חייב לעבור אימות HMAC. SPF/DKIM verification אוטומטי. Rate limit של 50 מיילים/יום למשתמש. כתובות לא מוכרות עוברות סיווג Gemini לפני עיבוד.

סקירה כללית

Double Check פותרת בעיה כואבת אחת: כשמזמינים סופר/דברים אונליין, חצי מהפעמים יש פערים בין מה ששילמת למה שהגיע. פריטים חסרים, כמויות שגויות, מוצרים מוחלפים, פגי תוקף. רוב האנשים מוותרים על הפיצוי כי זה מסובך מדי לעקוב.

האפליקציה מקצרת את התהליך לדקה: מצלמים את הקבלה (או מעבירים מייל אישור הזמנה) → המערכת מחלצת אוטומטית את הפריטים → במהלך הפריקה מסמנים בצ׳קליסט מה הגיע → מקבלים מייל מוכן לשליחה לשירות הלקוחות עם רשימת הפערים.

✨ KILLER FEATURE Inbox Magic — לכל משתמש יש כתובת מייל ייחודית (כמו swift-eagle-42@inbox.doublecheck.app). מעבירים אליה אישור הזמנה מאמזון / AliExpress / SHEIN / רמי לוי / שופרסל — וההזמנה נכנסת אוטומטית. בלי צילום, בלי OCR, בלי הקלדה. זה פותח את כל ההזמנות מחו״ל בלי עבודה נוספת ובונה moat דאטה (אנחנו רואים את כל ההזמנות, יודעים מי ספק בעייתי).

Value Proposition

  • מהיר — צילום אחד מחליף הקלדה ידנית של 30+ פריטים.
  • מדויק — צ׳קליסט מובנה מונע "לשכוח לבדוק" פריט.
  • מוכן לשליחה — דוח הפערים מנוסח בעברית מקצועית, מוכן להעתקה למייל.
  • היסטורי — שומר את כל ההזמנות, רואים תבניות של ספקים בעייתיים.
  • גלובלי — באמצעות Inbox Magic, האפליקציה תומכת באמזון, AliExpress, SHEIN, Temu, iHerb ועוד.
  • Multi-currency — המרה אוטומטית בין מטבעות (USD/EUR/GBP/ILS) לפי תאריך החיוב.

Target Users

  • צרכנים פרטיים — מזמינים סופר אונליין (רמי לוי, שופרסל, וולט מרקט וכו׳)
  • קונים בחו״ל — Amazon, AliExpress, SHEIN — דרך Inbox Magic, ללא צילומי קבלה.
  • בעלי עסקים קטנים — בתי קפה, מסעדות, חנויות שמקבלים משלוחים מספקים.
  • מנהלי משק בית — אחראים על הזמנות חודשיות עבור משפחה/דירת שותפים.

תהליך עבודה (User Flow)

  1. פתיחת האפליקציה המשתמש פותח את ה-PWA (אייקון במסך הבית או דרך הדפדפן). מסך ראשי: כפתור גדול "צלם קבלה" + רשימת משלוחים אחרונים.
  2. צילום / העלאת קבלה המצלמה נפתחת ישירות (mobile) או דיאלוג קובץ (desktop). תומך ב-JPG/PNG/HEIC/PDF. אפשר לצלם כמה דפים אם הקבלה ארוכה. לפני שליחה — preview עם אפשרות crop / סיבוב / מחיקה.
  3. חילוץ אוטומטי (OCR) התמונה נשלחת ל-API שמריץ Gemini Vision עם prompt מובנה. מחזיר JSON: שם החנות, תאריך, רשימת פריטים (שם, כמות, מחיר, סוג מידה). זמן ממוצע: 5-10 שניות. במהלך זה — Loading state עם progress bar.
  4. סקירה ועריכה המשתמש רואה את הפריטים שחולצו בטבלה עריכה. אפשר: לתקן שם, לשנות כמות, להוסיף פריט שלא זוהה, למחוק פריט שגוי. כפתור "המשך לסימון" כשהכל תקין.
  5. סימון משלוח (Live Tracking) מסך הצ׳קליסט הראשי — כל פריט מופיע עם תיבת סימון (✓), שדה "כמות שהגיעה" (אם פחות מהמוזמן), ותגיות מהירות: ✅ הגיע במלואו / ⚠️ חלקי / ❌ לא הגיע / 💔 פגום. המשתמש מסמן תוך כדי פריקת המשלוח.
  6. סיכום פערים אחרי שמסיימים — לוחצים "סיים בדיקה". המערכת מציגה דוח סיכום: כמה פריטים תקינים, כמה חסרים, סך החיוב המגיע להחזר. כפתור גדול: "הכן מייל לשירות לקוחות".
  7. הכנת מייל מסך עם תצוגה מקדימה של המייל בעברית מקצועית, כולל מספר הזמנה, תאריך, רשימת פערים מסודרת, וסכום ההחזר המבוקש. שלושה כפתורים: "העתק", "פתח באפליקציית מייל", "שתף בוואטסאפ".
  8. שמירה בהיסטוריה המשלוח נשמר אוטומטית. אפשר לחזור אליו, לראות את הפערים שהיו, לעקוב אם התקבל החזר (סטטוס: pending / refunded / rejected).
UX Goal מצילום ועד מייל מוכן לשליחה — פחות מ-90 שניות. המשתמש בכלל לא צריך להקליד את שמות הפריטים. רק לסמן ולשלוח.

מסכי המערכת

12 מסכים מרכזיים (9 בסיסיים + 3 ל-Inbox Magic). כולם חייבים לתמוך ב-RTL, mobile-first, dark mode (אופציונלי לגרסה 1.5).

SCREEN 01
Home / Dashboard
CTA ראשי "צלם קבלה חדשה" + רשימת המשלוחים האחרונים עם סטטוס (בבדיקה / סיים / החזר התקבל).
SCREEN 02
Capture / Upload
פתיחת מצלמה ישירה, אפשרות גלריה / קובץ. תומך multi-page (כמה תמונות לקבלה אחת). preview + crop.
SCREEN 03
OCR Loading
מסך מעבר עם אנימציה ו-progress states ("מנתח תמונה" / "מחלץ פריטים" / "מסיים..."). זמן ממוצע 5-10 שניות.
SCREEN 04
Review & Edit Items
טבלת פריטים שחולצו. עריכה inline של שם/כמות/מחיר. כפתור "+ הוסף פריט". Validation על מספרים.
SCREEN 05
Tracking Checklist
המסך הראשי — צ׳קבוקס לכל פריט, ארבעה סטטוסים (תקין/חלקי/חסר/פגום), שדה כמות שהגיעה, מונה התקדמות חי.
SCREEN 06
Discrepancy Summary
דוח סיום: 3 KPIs (תקין/חסר/פגום), סכום החזר משוער, רשימת כל הפערים מקובצת לפי סוג.
SCREEN 07
Email Composer
תצוגת מייל מנוסחת. שדה נמען (אופציונלי), עריכת הטקסט, כפתורי שיתוף (mailto / WhatsApp / העתק).
SCREEN 08
Delivery History
רשימת כל המשלוחים. סינון לפי חנות / סטטוס / תאריך. סטטיסטיקה: סה״כ ספקים בעייתיים, % משלוחים מושלמים.
SCREEN 09
Settings & Account
חיבור חשבון Google (אופציונלי לסנכרון), העדפות (שפה, פורמט מייל), ניהול נתונים (ייצוא/מחיקה).
SCREEN 10 ✨ NEW
Inbox Magic Setup
📨 הצגת כתובת ה-Inbox האישית עם Copy + QR code + הוראות העברת מיילים מ-Gmail/Outlook + כפתור "שלח מייל בדיקה".
SCREEN 11 ✨ NEW
Email Intakes Inbox
📨 רשימת כל המיילים שנקלטו: סטטוס פיענוח (✅ הצליח / ⚠️ חלקי / ❌ לא זוהה כהזמנה), כפתור "פעל ידנית" לתיקון.
SCREEN 12 ✨ NEW
Foreign Order View
📨 תצוגה ייחודית להזמנות חו״ל: מטבע מקור + ILS, ETA למשלוח, מע״מ/מכס נפרדים, שערי המרה היסטוריים.
Design Principle מסך 5 (Tracking) הוא ה-MVP. אם המשתמש לא יסיים את המשלוח שם — שום דבר אחר לא משנה. הוא חייב להיות הכי נוח, הכי מהיר, הכי ברור — מותאם לסיטואציה של "אדם עם 30 שקיות מסביב".

מודל נתונים (Supabase / Postgres)

סכמה מינימלית של 4 טבלאות. כל טבלה כוללת RLS (Row Level Security) — משתמש יכול לגשת רק למשלוחים שלו.

-- ╔═══ user_inboxes ═══╗ (NEW — Inbox Magic)
CREATE TABLE user_inboxes (
  user_id          UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  inbox_local      TEXT UNIQUE NOT NULL,         -- "swift-eagle-42"
  inbox_full       TEXT GENERATED ALWAYS AS           -- "swift-eagle-42@inbox.doublecheck.app"
                   (inbox_local || '@inbox.doublecheck.app') STORED,
  created_at       TIMESTAMPTZ DEFAULT now(),
  last_email_at    TIMESTAMPTZ,
  total_emails     INTEGER DEFAULT 0,
  is_active        BOOLEAN DEFAULT true          -- false אם המשתמש איפס את הכתובת
);

-- ╔═══ deliveries ═══╗
CREATE TABLE deliveries (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  source          TEXT DEFAULT 'photo',           -- photo | email | extension | manual
  store_name      TEXT,                        -- "רמי לוי", "Amazon", "AliExpress"
  vendor_id       TEXT,                        -- internal vendor key: "amazon_us", "rami_levy"
  order_number    TEXT,
  order_date      DATE,
  delivery_date   DATE,
  expected_eta    DATE,                        -- ETA from email (foreign orders)
  original_currency TEXT DEFAULT 'ILS',       -- USD/EUR/GBP/ILS/...
  original_total  DECIMAL(10,2),               -- בסכום המקורי
  exchange_rate   DECIMAL(10,4),               -- שער המרה לתאריך ההזמנה
  total_amount    DECIMAL(10,2),               -- מחושב ב-ILS תמיד
  vat_amount      DECIMAL(10,2),
  customs_amount  DECIMAL(10,2) DEFAULT 0,
  receipt_url     TEXT,                        -- Supabase Storage path (אם photo)
  email_intake_id UUID REFERENCES email_intakes(id), -- אם source = email
  status          TEXT DEFAULT 'pending',        -- pending | reviewing | completed | refunded
  refund_status   TEXT,
  notes           TEXT,
  created_at      TIMESTAMPTZ DEFAULT now(),
  updated_at      TIMESTAMPTZ DEFAULT now()
);

-- ╔═══ email_intakes ═══╗ (NEW — Inbox Magic)
CREATE TABLE email_intakes (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID REFERENCES auth.users(id),
  vendor_id       TEXT,                        -- "amazon_us", "aliexpress", "unknown"
  from_address    TEXT,
  subject         TEXT,
  html_url        TEXT,                        -- Supabase Storage (raw HTML for debugging)
  parsed_status   TEXT,                        -- success | failed | not_an_order | spam
  parser_used     TEXT,                        -- "vendor_adapter:amazon_us" | "gemini_generic"
  confidence      DECIMAL(3,2),                -- 0.00-1.00
  error_msg       TEXT,
  received_at     TIMESTAMPTZ DEFAULT now(),
  processed_at    TIMESTAMPTZ
);

-- ╔═══ delivery_items ═══╗
CREATE TABLE delivery_items (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  delivery_id     UUID REFERENCES deliveries(id) ON DELETE CASCADE,
  position        INTEGER,                     -- מיקום בקבלה (1, 2, 3...)
  sku             TEXT,                        -- ברקוד / מק״ט אם קיים
  name            TEXT NOT NULL,
  unit            TEXT,                        -- "ק״ג", "יח׳", "ליטר"
  ordered_qty     DECIMAL(10,3),               -- כמות שהוזמנה לפי הקבלה
  unit_price      DECIMAL(10,2),
  total_price     DECIMAL(10,2),
  received_qty    DECIMAL(10,3),               -- כמות שהגיעה בפועל (NULL = טרם נבדק)
  status          TEXT,                        -- ok | partial | missing | damaged | wrong_item
  issue_note      TEXT,                        -- "פג תוקף", "דחוס", "החליפו לי במותג אחר"
  photo_url       TEXT,                        -- ראיה לפגם (אופציונלי)
  created_at      TIMESTAMPTZ DEFAULT now()
);

-- ╔═══ user_preferences ═══╗
CREATE TABLE user_preferences (
  user_id             UUID PRIMARY KEY REFERENCES auth.users(id),
  default_email       TEXT,                    -- מייל ברירת מחדל לשירות לקוחות
  email_signature     TEXT,                    -- חתימה מותאמת אישית
  notification_email  BOOLEAN DEFAULT true,
  language            TEXT DEFAULT 'he'
);

-- ╔═══ store_profiles ═══╗ (lookup table)
CREATE TABLE store_profiles (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name                TEXT UNIQUE NOT NULL,        -- "רמי לוי", "שופרסל"
  support_email       TEXT,                    -- אימייל שירות לקוחות ידוע
  support_phone       TEXT,
  refund_form_url     TEXT                     -- אם יש טופס מקוון
);

-- ╔═══ Row Level Security ═══╗
ALTER TABLE deliveries ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users_own_deliveries" ON deliveries
  USING (auth.uid() = user_id);

ALTER TABLE delivery_items ENABLE ROW LEVEL SECURITY;
CREATE POLICY "users_own_items" ON delivery_items
  USING (delivery_id IN (SELECT id FROM deliveries WHERE user_id = auth.uid()));

חילוץ פריטים (OCR Pipeline)

החלק הקריטי ביותר באפליקציה. אם החילוץ לא מדויק — המשתמש יזרוק את האפליקציה אחרי הניסיון השני. לכן ה-pipeline הוא 3 שלבים עם בקרת איכות.

Stage 1: Pre-processing (Client-side)

  • אם תמונה גדולה מ-2MB — דחיסה ל-1920px max (canvas API).
  • HEIC → JPEG conversion עם heic2any.
  • PDF multi-page → רשימת תמונות (PDF.js render).
  • זיהוי אוטומטי של זווית הקבלה ויישור (אופציונלי v2).

Stage 2: Vision API Call

קוראים ל-Gemini 2.5 Flash עם prompt מובנה ו-responseSchema. זה מבטיח שתמיד נקבל JSON תקני, גם אם המודל "מתעצל".

Stage 3: Validation & Confidence Score

  • Math check — sum of all total_price ± 5% from total_amount. אם לא — דגל.
  • Sanity check — האם כל unit_price × ordered_qty ≈ total_price?
  • Confidence calculation — מספר הפריטים שעברו את ה-checks חלקי סך הפריטים.
  • אם confidence < 0.7 → ה-UI מציג אזהרה: "ייתכנו אי-דיוקים, אנא בדוק את הרשימה לפני השלב הבא".
חשוב תמונות עם רזולוציה נמוכה (טלפון ישן, תאורה גרועה) — דיוק ירד ל-70-80%. הפתרון: מסך עזרה עם טיפים לצילום (אור טוב, ישר מלמעלה, רקע נקי). אפשר גם להציע "צלם שוב" אם confidence נמוך מאוד.

Inbox Magic — קליטת הזמנות במייל 📨

זה ה-Killer Feature של האפליקציה. במקום לצלם קבלה — המשתמש פשוט מעביר לכתובת מייל ייחודית את אישור ההזמנה (Order Confirmation) שקיבל מאמזון / AliExpress / SHEIN / רמי לוי / כל ספק. המערכת מפענחת את המייל אוטומטית ויוצרת משלוח מוכן למעקב.

למה זה משנה הכל אפס חיכוך — אין צילום, אין OCR, אין הקלדה. גלובלי — עובד על כל ספק בעולם, לא רק ספקים מקומיים. Moat דאטה — המערכת רואה את כל ההזמנות של כל המשתמשים, בונה profile של ספקים בעייתיים. Onboarding 10x — המשתמש מתחיל להפיק ערך תוך 30 שניות במקום לחכות למשלוח הבא.

User Flow

  1. קבלת inbox אישי בהרשמה — המערכת מייצרת לכל משתמש כתובת ייחודית בפורמט [adjective]-[animal]-[N]@inbox.doublecheck.app, למשל swift-eagle-42@inbox.doublecheck.app. ניתן לזכור, בטוח, אנונימי.
  2. הצגה במסך הגדרות במסך Settings → "Inbox Magic" — המשתמש רואה את הכתובת + כפתור Copy + QR code לסריקה מהמחשב + כפתור "שתף בוואטסאפ לעצמי".
  3. העברת מייל הזמנה המשתמש מקבל אישור הזמנה מהחנות. לוחץ Forward → מזין את הכתובת האישית → שולח. או: מגדיר Filter ב-Gmail שמעביר אוטומטית כל מייל מ-auto-confirm@amazon.com לכתובת האישית.
  4. קליטה אוטומטית (3-5 שניות) Cloudflare Email Worker קולט את המייל, מפרסר את ה-MIME, מזהה את הספק לפי From:, ושולח את גוף ה-HTML לפרסר ה-Gemini. במקרה של ספק מוכר — משתמש ב-vendor adapter ייעודי לדיוק של 99%.
  5. Push notification "📨 הזמנה חדשה זוהתה — Amazon, $87.50, 12 פריטים. הקש לעיון" — המשתמש לוחץ ועובר ישר למסך Review.
  6. אישור והפיכה למעקב המשתמש בודק שהפריטים נכונים (90%+ זה מושלם), לוחץ "אשר" — וההזמנה מוכנה. בזמן שהמשלוח מגיע (יום / שבוע / חודש בחו״ל), היא יושבת ב-"בהמתנה". כשמסמנים שהמשלוח הגיע — עוברים לצ׳קליסט הסימון.

Cloudflare Email Worker (Code)

// /workers/inbox-magic/src/index.ts
import PostalMime from "postal-mime";

export default {
  async email(message: EmailMessage, env: Env) {
    // 1. Parse MIME
    const raw = await streamToArrayBuffer(message.raw, message.rawSize);
    const parser = new PostalMime();
    const parsed = await parser.parse(raw);

    // 2. Extract user from inbox address
    const inboxLocal = message.to.split("@")[0];  // "swift-eagle-42"
    const user = await env.SUPABASE.from("users").select().eq("inbox_local", inboxLocal).single();
    if (!user) return message.setReject("Unknown inbox");

    // 3. Spam protection — accept only if from known retailer OR Gemini classifies as order
    const sender = parsed.from?.address ?? "";
    const isKnownRetailer = matchVendor(sender);

    // 4. Forward to Vercel webhook (signed with HMAC)
    await fetch("https://double-check.vercel.app/api/inbox/webhook", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "X-Inbox-Signature": await sign(env.WEBHOOK_SECRET, payload),
      },
      body: JSON.stringify({
        user_id: user.id,
        vendor: isKnownRetailer ?? "unknown",
        subject: parsed.subject,
        html: parsed.html,
        text: parsed.text,
        received_at: new Date().toISOString(),
      }),
    });
  },
};

Vercel Webhook Handler (Code)

// /app/api/inbox/webhook/route.ts
export async function POST(req: Request) {
  // 1. Verify HMAC signature (reject forged webhooks)
  const sig = req.headers.get("X-Inbox-Signature");
  const body = await req.text();
  if (!verifyHmac(sig, body, process.env.WEBHOOK_SECRET!)) {
    return new Response("Forbidden", { status: 403 });
  }

  const { user_id, vendor, html, text, subject } = JSON.parse(body);

  // 2. Run vendor adapter OR Gemini fallback
  const adapter = VENDOR_ADAPTERS[vendor];
  const extracted = adapter
    ? await adapter.parse({ html, text, subject })
    : await extractWithGemini({ html, text, subject });

  // 3. Currency conversion if foreign
  if (extracted.currency !== "ILS") {
    const rate = await getExchangeRate(extracted.currency, "ILS", extracted.order_date);
    extracted.exchange_rate = rate;
    extracted.ils_total = extracted.original_total * rate;
  }

  // 4. Save to Supabase + send push notification
  const delivery = await saveDelivery(user_id, { ...extracted, source: "email" });
  await sendPush(user_id, { title: "📨 הזמנה חדשה זוהתה", delivery_id: delivery.id });

  return Response.json({ ok: true, delivery_id: delivery.id });
}

Spam & Abuse Protection

  • Whitelist of known retailers — אם ה-From: מתאים לרשימת ספקים ידועים, מקבלים אוטומטית.
  • Gemini classifier — לכתובת לא מוכרת, Gemini מקבל את המייל ומחזיר is_order_confirmation: true|false. רק אם true — ממשיכים.
  • Rate limiting — מקס׳ 50 מיילים/יום למשתמש (חוסם botnet abuse).
  • SPF/DKIM verification — Cloudflare מבצעת את זה אוטומטית. מיילים falsified נדחים.
  • Disposable inbox rotation — המשתמש יכול לאפס את הכתובת בלחיצה (אם spam).

דוח פערים — Email Generation

זה ה-Killer Feature. אחרי שהמשתמש סיים לסמן — הוא צריך להעתיק טקסט מקצועי, מסודר, ומוכן לשליחה למייל / וואטסאפ. הטקסט נוצר אוטומטית מהנתונים — אין צורך ב-AI generation, רק template מילולי חכם.

Template Structure

// pseudocode of the generation logic
function generateEmail(delivery: Delivery, items: Item[]) {
  const missing  = items.filter(i => i.status === "missing");
  const partial  = items.filter(i => i.status === "partial");
  const damaged  = items.filter(i => i.status === "damaged");
  const wrong    = items.filter(i => i.status === "wrong_item");

  const refund =
    missing.reduce((s, i) => s + i.total_price, 0) +
    partial.reduce((s, i) => s + i.unit_price * (i.ordered_qty - i.received_qty), 0) +
    damaged.reduce((s, i) => s + i.total_price, 0);

  return {
    subject: `פערים בהזמנה ${delivery.order_number} מתאריך ${delivery.order_date}`,
    body: renderTemplate({ delivery, missing, partial, damaged, wrong, refund }),
    refund_amount: refund,
  };
}
